SabbathCue is a Tauri v2 desktop app, so you'll need the standard Tauri toolchain
plus a Python interpreter for the data-prep pipeline. The complete list:

| Tool                                                                                     | Why you need it                                                                                               |
| ---------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| [Bun](https://bun.sh)                                                                    | JavaScript runtime + package manager for `web/`, build scripts, and the orchestrator                          |
| [Rust](https://rustup.rs) (stable, 1.77.2+)                                              | Compiles the Tauri backend and Rust workspace crates                                                          |
| [Tauri v2 prerequisites](https://v2.tauri.app/start/prerequisites/)                      | Platform-specific system libraries (WebView2 on Windows, WebKit on Linux)                                     |
| [Python 3](https://www.python.org/)                                                      | Downloads copyrighted translations, exports the embedding model to ONNX, and runs the development Vosk worker |
| [LibreOffice](https://www.libreoffice.org/) (optional)                                   | Converts `.ppt`/`.pptx` decks to slides for service-plan and theme imports                                    |
| [Deepgram](https://deepgram.com/) or [Soniox](https://soniox.com/) API key (optional)    | Cloud speech-to-text, only if you don't want to run local Vosk                                                |

<Callout title="Local vs Cloud STT" type="info">
  You only need **one** speech-to-text engine. Vosk runs locally with no API
  costs and is the default. Deepgram and Soniox stream to cloud APIs and need
  paid API keys. See [Speech-to-text](/docs/getting-started/speech-to-text) for
  tradeoffs.
</Callout>

<Callout title="PowerPoint import" type="info">
  Importing PowerPoint decks (sermon slides or theme backgrounds) requires a
  local [LibreOffice](https://www.libreoffice.org/) install. SabbathCue finds
  `soffice` on your `PATH` or the default install location; set
  `SABBATHCUE_SOFFICE_PATH` to override it.
</Callout>

## Hardware

SabbathCue runs comfortably on modern laptops:

- **CPU:** anything from the last five years; a recent Apple Silicon or
  Ryzen/Intel laptop will keep the audio capture, STT, and detection
  pipeline well under a single core.
- **RAM:** 8 GB minimum, 16 GB recommended if you precompute embeddings on
  CPU (the GPU path uses far less memory).
- **GPU:** optional. The setup script auto-detects CUDA/MPS for embedding
  precomputation and falls back to ONNX CPU otherwise.
- **Disk:** ~3 GB after `setup:all` finishes (Bible data, embeddings, ONNX
  models).

## Operating system

| OS                                  | Status                                                                                                     |
| ----------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| macOS 13+ (Apple Silicon and Intel) | First-class support                                                                                        |
| Windows 10/11 (x86_64)              | First-class support - see [Platform setup](/docs/getting-started/platform-setup#windows) for MSVC guidance |
| Linux (Debian/Ubuntu/Arch)          | First-class support - needs the Tauri WebKitGTK system packages                                            |

Once these are installed, head to the [installation guide](/docs/getting-started/installation).
