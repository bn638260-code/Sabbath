SabbathCue supports two STT engines, and you can swap between them at runtime
from the settings dialog. They map cleanly to two different operating
modes: zero-marginal-cost local inference, or offload-to-the-cloud
streaming.

## Option 1: Whisper (local, default)

Whisper is an offline speech recognition model that transcribes audio in
real time on your machine. SabbathCue ships with the smallest, lightest
model — `ggml-tiny.en.bin` — so it runs comfortably without a GPU. A
Bible-reference prompt biases recognition toward book names, chapter and
verse numbers, and common scripture vocabulary.

To install the Whisper model:

1. Run `bun run download:whisper` to fetch `ggml-tiny.en.bin` into
   `models/whisper/`
2. Or set `SABBATHCUE_WHISPER_MODEL` to point to an existing GGML model file

The model is required before Whisper transcription starts. The app shows
model installation status in Settings → Speech Recognition.

<Callout title="Vosk compatibility" type="info">
Earlier builds used Vosk as the local engine. Vosk remains available for
backward compatibility, but Whisper is now the default local provider and
existing Vosk selections migrate to Whisper automatically.
</Callout>

## Option 2: Deepgram

Deepgram is a cloud speech-to-text service that streams audio over a
WebSocket. SabbathCue also has a REST fallback that re-uploads short windows if
the WebSocket disconnects.

### 1. Get an API key

Sign up at [deepgram.com](https://deepgram.com/) and copy your API key.

### 2. Add it to `.env`

Create a `.env` file in the project root:

```bash
DEEPGRAM_API_KEY=dg_your_key_here
```

Restart `bun run tauri dev` so the new key is picked up.

### 3. Switch to Deepgram in the app

Open Settings → STT and choose **Deepgram**. The status indicator shows
WebSocket health in real time; if it falls back to REST, you'll see that
labeled too.

## Switching engines at runtime

Both engines are wired into the same transcript stream, so you can switch
without restarting the session. The active translation, sermon context,
and queue state are preserved across the swap.

## Comparing the two

| | Whisper (local) | Deepgram (cloud) |
| --- | --- | --- |
| API cost | Free | Per-minute billing |
| Internet required | No, after the model is installed | Yes (WebSocket) |
| Recognition mode | Offline neural model + Bible prompt | Keyword-boosted streaming |
| Transcript coverage | General speech | Full general speech |
| Punctuation | Basic | Excellent |
| Works offline | Yes | No |

Whisper runs the tiny.en model entirely on your machine, so it works
offline with no per-minute cost. For the best full-sermon transcription
quality on a reliable network, Deepgram remains the recommended option.
Pick Whisper for the no-cost offline path, Deepgram for maximum accuracy.
